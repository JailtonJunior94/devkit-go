# Unit Test Conventions

This document defines the unit testing conventions for `devkit-go`. Follow these standards across all packages in `pkg/`.

## Framework

| Tool | Role |
|---|---|
| `testing` (stdlib) | Test runner and benchmarks |
| `github.com/stretchr/testify/require` | Assertions with immediate failure |
| `github.com/stretchr/testify/suite` | Shared setup/teardown via `SetupTest` |
| `github.com/stretchr/testify/mock` | Expectations on generated mocks |

Use `require` (not `assert`) as the default assertion package. `require` stops the test immediately on failure, producing cleaner output when multiple assertions follow a precondition.

Use `suite` only when multiple test cases share a non-trivial setup or teardown. For simple cases, plain `TestXxx` functions with `require` are preferred.

## Structure: AAA

Every test follows **Arrange → Act → Assert**:

```go
func TestUnitOfWork_Do_CommitsOnSuccess(t *testing.T) {
    // Arrange
    mockMgr := managermocks.NewMockManager(t)
    mockTx  := dbmocks.NewMockTx(t)
    // ... configure expectations

    uow := uow.New[int](mockMgr)

    // Act
    result, err := uow.Do(context.Background(), func(_ context.Context, tx database.DBTX) (int, error) {
        return 42, nil
    })

    // Assert
    require.NoError(t, err)
    require.Equal(t, 42, result)
}
```

## Table-Driven Tests

Use table-driven tests for logic with multiple input/output variations:

```go
func TestPostgresConfig_Validate(t *testing.T) {
    tests := []struct {
        name    string
        cfg     postgres.PostgresConfig
        wantErr bool
    }{
        {
            name:    "valid DSN",
            cfg:     postgres.PostgresConfig{DSN: "postgres://user:pass@host/db"},
            wantErr: false,
        },
        {
            name:    "missing host and DSN",
            cfg:     postgres.PostgresConfig{User: "user", Database: "db"},
            wantErr: true,
        },
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            err := tt.cfg.Validate()
            if tt.wantErr {
                require.Error(t, err)
            } else {
                require.NoError(t, err)
            }
        })
    }
}
```

## Test Naming

Names must be in **English** and describe observable behaviour, not implementation:

```
Test<Subject>_<Method>_<ExpectedOutcome>
```

Examples:

- `TestUnitOfWork_Do_RollsBackOnPanic`
- `TestUnitOfWork_Do_RollsBackOnError`
- `TestUnitOfWork_Do_ReturnsErrNestedTransaction`
- `TestManager_Shutdown_IdempotentOnDoubleCall`
- `TestPostgresConfig_Validate_ReturnsMissingFieldErrors`
- `TestFromContext_ReturnsFalseOnEmptyContext`

## Mocks

Mocks are generated by **mockery v3** from `.mockery.yml`. Never write mocks by hand.

Generated location: `pkg/<pkg>/mocks/` — one file per interface.

Import pattern:

```go
import (
    dbmocks      "github.com/JailtonJunior94/devkit-go/pkg/database/mocks"
    managermocks "github.com/JailtonjUnior94/devkit-go/pkg/database/manager/mocks"
    migmocks     "github.com/JailtonJunior94/devkit-go/pkg/database/migration/mocks"
    o11ymocks    "github.com/JailtonJunior94/devkit-go/pkg/observability/mocks"
)
```

Always pass `t` to the mock constructor so that mockery registers cleanup automatically:

```go
m := managermocks.NewMockManager(t)
```

Set call count expectations explicitly when the count matters:

```go
m.On("Ping", mock.Anything).Return(nil).Once()
```

## Race Detector: `-race` is Mandatory

Every unit test run must include the race detector. The project's `make test` target already enforces this:

```bash
go test -v -race -coverprofile=coverage.txt -covermode=atomic ./...
```

Do **not** skip `-race` in local runs. Tests that pass without `-race` but fail with it are broken by definition.

## Isolation

- No shared mutable state between test cases. Each `t.Run` gets its own variables.
- `SetupTest` in suites resets all mocks and state before every case.
- No real network or database calls in unit tests. Use mocks for `Manager`, `DBTX`, `Migrator`, and `Observability`.
- No timing assumptions. Use explicit synchronisation (`chan`, `sync.WaitGroup`) when goroutines are involved.

## Benchmarks

Benchmarks live alongside unit tests in `*_bench_test.go` (or `*_test.go` with `Benchmark` prefix). Always call `b.ResetTimer()` after setup and `b.ReportAllocs()` to surface allocation counts.

```go
func BenchmarkUoW_Do_Commit(b *testing.B) {
    // setup outside timer
    b.ResetTimer()
    b.ReportAllocs()
    for i := 0; i < b.N; i++ {
        // measured operation
    }
}
```

Reference: `pkg/observability/otel/tracer_bench_test.go` for a benchmark that validates allocation counts in hot paths.

## Regenerating Mocks

```bash
make mocks-reset   # clean + regenerate all mocks
make mocks-clean   # remove generated files only
```

Run `make mocks-reset` after any interface change in `pkg/database` or `pkg/observability`.
